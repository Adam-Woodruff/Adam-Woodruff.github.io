RSA encryption

12/11/2021

To Aid in my computer science project in my spare time I decided to endeavour in programming my version of the widely used, RSA encryption algorithm. To get a basic idea of how I could go about doing this I watched videos of the maths behind how it works. This I found to be very useful particularly the videos from Eddie Woo.

After gaining the knowledge of how the algorithm works, I began piecing it together. Firstly, I started by generating the prime numbers used for the key. I did this by generating a random odd number checking to see if it was prime and if it wasn’t I added 2 and repeated until I found one. While this is most likely not the most efficient way it works in reasonable time for my use case. To check if the random number is prime, I used Fermats test for primality this test is thoroughly explained elsewhere on the internet but in general it is better to run the test multiple times as there is a small chance the test gives a false positive with specific numbers.

When calculating large powers, the most popular method of calculation is typically modular exponentiation. While there is a built-in python function to do this that is almost certainly faster, I wanted to program my own function just to fully understand how it works.

To generate the privatekey I followed the documented method as learnt from research. The public key must be between 1 and the number of co-primes of the two prime number generated while also being co-prime with the two primes number multiplied.

Example:

P = 11, q = 13

5 x 7 = 143

(11-1) (13-1) = 120

Φ(n) = 120

This shows us there are no common factors between 7, 143 and 120 making them co-prime.

1 < 7 < 120

So, 7 is our public key

The equation to find the private key makes use of both the public key and n so that it satisfies the equation

After the private key had been generated, I used the extended Euclidean algorithm to calculate the public key. This calculates the greatest common divisor of p and q. The result from this function was the working public key.

After the keys had been generated it is now just interpreting the input and encoding it using the keys. when using RSA encryption is the same process and decryption just with different keys, Because of this I made use of one function called conversion. This function used modular exponentiation to speed up the process of following the RSA formula for encryption and decryption.

Example:

Public key = (7, 143)

Plain text – Hello World
